More CSS

Notes

  • Vendor prefixes - add support to browsers for new CSS features (-webkit, -moz, -ms, -o)
    • New browsers no longer need them for CSS transitions
    • CSS animations are relatively new and need them
  • Page state - manages all the elements on a page
  • Box sizing is powerful for layout because when you're setting widths of things, you know it's just that width
  • By default, when you add padding and margin, the amt you set is added to the width of the content
  • There are two types: content-box and border-box, determines if you are adding margin and padding to the width number
  • Content-box is the implicit default value. Used to reset. ie) if you set all elements to be border-box but you need to override it for a special element
  • Border-box makes margin and padding part of the width value

3 March 2015

CSS Day Three

Notes

  • Shorthand CSS
    • margin: Top Right Bottom Left
    • margin: TB RL
    • margin: T RL B
  • Floated elements collapse their parent elements. ie) If a div contains only floated elements, the browser will think that the container is empty.
  • Browser doesn't care about the size of a child element unless it has to
  • 3 ways to fix float issues:
    • overflow:hidden; on parent
    • clearfix
    • Float the parent
  • Don't use heights on CSS elements except rarely (vertically center something, make a circle)
  • Overflow hidden - cropping content
  • Setting widths helps stabilize floats
    • You can set max-widths and min-widths for changing browser sizers
  • z-index specifies stack order of elements
  • HTML is also parsed down. An element overlapping another is going to implicitly go on top of the one that came before it
  • position: sticky is pretty cool. It says for the element to go where it naturally wants to until it reaches a certain point. Used on Tumblr dashboard (see users' avatar movement as you scroll). Not supported by all browsers, relatively new
  • vh and vw (viewport height/viewport width) are sizing properties you can use
  • position: fixed is based off the viewport
  • position: absolute inherits from parent

CSS Day Two

Notes

  • Relative units (percentages) in css behave differently than fixed units (pixels, points, inches, etc.) There's also relative fixed (ems). Relative units gain value by looking at parent element as it traverses up tree based on additional criteria.
  • The purpose of * selector is to reset default browser styles
  • No #ids ever - flexibility of CSS
  • Class names for everything
  • Elements used sparingly
  • Naming conventions - object oriented CSS
    • parent
    • parent-child
    • parent-child--child
  • Ordering CSS
    • positioning
    • box model
    • decorative
  • Inline elements take up the size of their content. They cannot accept height and width properties
  • margin: auto; centers block elements with a defined width

CSS Day One: Putting the Cascade in Cascading Style Sheets

Marking up cookies (with CSS)

Cookies

Notes

  • Specficity - how specific is your selector. A method of conflict resolution within the cascade. When two or more declarations that apply to the same element, and set the same property, have the same importance and origin, the declaration with the most specific selector will take precedence.
  • Order - CSS is parsed from top to bottom
  • Cascade - DOM. The cascade is a mechanism for determining which styles should be applied to a given element, based on the rules that have cascaded down from various sources.
  • Good CSS has low specificity (more generic, less specific, easier/faster to edit)
  • Specificity increases from top to bottom

Reimer's article convinced me how important having a process to consistently follow is for a web designer. Although I've had experience working with HTML and CSS most of my work has either been personal or working with people who I already had developed relationships with, and the article has helped me become more aware of the adjustments I would probably have to make in the real world working with clients. The updates that HTML5 introduced seem sensible and practical to me now that looking back at older versions of HTML makes me cringe. It's interesting that Allsopp is arguing that HTML5 has not reached the potential semantic capacity of HTML, and that there are still problems to be solved that HTML5 is not able to. PROGRAMMING IS NOT A CRAFT

People in technology have varying opinions on what software engineering entails. There are conflicts over whether or not software development is a craft and what exactly qualifies as a craft. In general however, the consensus follows that programming differentiates between those who are truly passionate and those who program for superficial reasons. I think programming is a craft because the process of software engineering involves design and creation. I also believe that the argument presented by Dan North relied heavily on his notion that most programmers are not masterful at their skills. Programming is a craft, but not all programmers are craftsmen.
