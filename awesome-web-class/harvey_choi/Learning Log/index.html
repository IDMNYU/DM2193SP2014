Following a Web Design Process.
How to Read a Wireframe.
Semantics in HTML5.

I found the article about following a Web Design Process interesting. Because I am taking the class Ideation and Prototyping, this process reminds me about all the important aspects to designing anything for the user. It must be easy to use, greet the user, interact with the user, and make the user feel like the web is person. Does this web interact friendly with the user?" These are the things to think about while developing a web.

My sixth learning log entry

Notes
  • On this day, we talked a lot about CSS and how to apply that in our index. We practiced coding CSS on a styles.css sheet. We tried different colors, headings, and fonts on our index html or the learning logs that we made. But mostly, we were talking about Order, Specificity, and Cascade. "Specificity is a type of weighting that has bearing on how your cascading style sheet (CSS) rules are displayed." (http://www.smashingmagazine.com/2007/07/27/css-specificity-things-you-should-know/). It is a selector to determine which rule could be applied to the same element in markup. Also the order of precedence is also important. For example, if a page contains a link to two style sheets, the rules in the second style sheet listed takes precedence over the rules in the first style sheet. We will learn more about this as the class continues.
